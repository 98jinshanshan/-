## 三、编排层：仍需补的通信总线(下阶段开发)
现状：`orchestrator.js` 用 `waitForSignal` 文件轮询(每2秒查文件),并打印"请打开 Codex 输入…",超时要"手动运行 node orchestrator.js --phase execute"。即三IDE之间靠人工搬运指令。

补全项(按优先级)：
1. **真通信层**：把文件轮询替换为 MCP server / HTTP 调用，让编排器能自动拉起 Codex/OpenClaw/Cursor 并回收结果。
2. **常驻编排进程 + 各IDE Agent 常驻监听**：替代"请打开 X 输入 Y"的人工步骤。
3. **Phase 1–7 全链驱动**：意图锚定→拆解→执行→审核→交付串成一条自动流水，闸门(Phase 8)作为终检。
4. **触发器**：单一入口(一句话)→自动生成 spec→自动分派→自动审核→自动交付+闸门。
