# 📖 README_FIRST：三条阅读路线

> **你是一个新 Agent，刚刚加入这个项目。不知道该从哪里开始？本文件为你制定三条阅读路线。**
>
> 选择最适合你的路线，按顺序读下去，不用读整个 Hub。

---

## 🚀 **路线 A：快速启动（10 分钟，适合想快速开始的 Agent）**

**目标**：理解"一句话输入 → 全自动执行 → 开发完成"的核心机制

**阅读顺序**：

### 第 1 步（3 分钟）：读总纲导言
📄 **文件**：`开发总纲_人类可读版.md`  
📍 **范围**：前 100 行（标题到"核心契约"章节）  
🎯 **获得**：理解"什么是这个开发系统"

```
学到的内容：
- 你输入一句话 → Agent 自问自答 44 问
- Agent 自动进行"去程拆解"和"回程合成"
- 监督者全程看着，确保不会跑偏（防梨树嫁接）
- 8 个 Phase + 8 个检查点
```

### 第 2 步（4 分钟）：读双程递归模型
📄 **文件**：`开发总纲_人类可读版.md`  
📍 **范围**：第 115-156 行（"双程递归模型"章节）  
🎯 **获得**：理解"去程"和"回程"是什么

```
学到的内容：
- 去程 Phase 1-5：设计。不写代码。
  Phase 1 意图锚定 → Phase 2 路径 → Phase 3 节点 → Phase 4 接口 → Phase 5 递归拆解
- 回程 Phase 6-8：实现。按照去程的蓝图写代码。
  Phase 6 实现集成 → Phase 7 通路验证 → Phase 8 交付收尾
- 每个 Phase 完成后，监督者检查一遍（检查点 0-7）
```

### 第 3 步（3 分钟）：理解你在这个流程中的角色
📄 **文件**：`AGENT_INSTRUCTION.md`  
📍 **范围**：第 10-20 行（角色定义表）+ 第 34-70 行（门禁引擎）  
🎯 **获得**：知道"你要做什么"

```
学到的内容：
- 你是 Codex / OpenClaw / Cursor 中的一个
- 你只负责你那个角色的工作
- 每个 Phase 必须产出一个文件（gate_file → produce_file 链）
- 不能跳 Phase
```

**✅ 快速启动完成！** 现在你知道了整个系统的"大框架"。可以开始做任务了。

---

## 📚 **路线 B：完整理解（1 小时，适合想完全掌握的 Agent）**

**目标**：理解每个 Phase 的具体做什么，以及检查点的校验逻辑

**在完成路线 A 的基础上，继续阅读**：

### 第 4 步（15 分钟）：学习 Phase 门禁引擎
📄 **文件**：`开发总纲_人类可读版.md`  
📍 **范围**：第 271-320 行（"Phase 门禁引擎"整章）  
🎯 **获得**：理解"为什么每个 Phase 都是强制的，不能跳"

```
学到的内容：
- gate_file：进入一个 Phase 的前置条件（前一个 Phase 的产物）
- produce_file：这个 Phase 必须产出什么文件
- 反跳过机制：如果某个 gate_file 不存在，直接 BLOCK，回退到缺失的最早 Phase
- 创作类简化规则：某些任务可以简化递归深度
```

### 第 5 步（20 分钟）：学习 8 个 Phase 的具体定义
📄 **文件**：`开发总纲_人类可读版.md`  
📍 **范围**：第 323-422 行（"8阶段定义"整章）  
🎯 **获得**：理解每个 Phase 具体做什么

```
学到的内容：
- Phase 1（意图锚定）：调用 44 问自审清单，自问自答，产出 spec.json
- Phase 2（路径锚定）：画出从起点到终点的路径图（path.json）
- Phase 3（节点识别）：识别路径上的所有节点（nodes.json）
- Phase 4（接口契约）：定义节点之间怎么连（interfaces.json）
- Phase 5（递归拆解）：对每个子节点，重新走 Phase 1-5
- Phase 6（实现集成）：按蓝图写代码
- Phase 7（通路验证）：写测试，验证通路
- Phase 8（交付收尾）：更新索引，标记完成
```

### 第 6 步（15 分钟）：学习 44 问自审清单
📄 **文件**：`开发总纲_问答题框架.md`  
📍 **范围**：第 60-223 行（"维度 1-8"的完整定义）  
🎯 **获得**：理解 Phase 1 的"44 问"具体是什么

```
学到的内容：
- 维度 1：意图锚定（5 问）→ 用户到底要什么？
- 维度 2：边界划定（5 问）→ 什么做、什么不做？
- 维度 3：架构建模（5 问）→ 数据从哪来、到哪去？
- 维度 4：接口契约（5 问）→ 模块之间怎么连？
- 维度 5：模块拆解（5 问）→ 拆到多细？
- 维度 6：实现路径（5 问）→ 先做什么、后做什么？
- 维度 7：验证闭环（7 问）→ 怎么验证？
- 维度 8：交付收尾（7 问）→ 怎么交付？
总共 44 个问题，不能压成 8 个摘要
```

### 第 7 步（10 分钟）：学习三 IDE 通信协议
📄 **文件**：`三IDE角色自动化联通协议_V1.0.md`  
📍 **范围**：第 6-99 行（"核心问题"到"触发文件协议"章节）  
🎯 **获得**：理解 Codex、OpenClaw、Cursor 怎么互相协作

```
学到的内容：
- 三个 IDE 实例独立运行，共享同一个 /workspace/ 目录
- 通过文件系统异步通信：A 写文件 → B 轮询发现 → B 执行 → B 写结果 → A 读取
- .pending → .processing → .result 三层状态机
- 详细的文件格式和目录结构
```

**✅ 完整理解完成！** 现在你可以理解任何一个 Phase，能自主判断自己该做什么。

## 🔬 **路线 C：深入研究（2 小时，适合想成为"总纲专家"的 Agent）**

**在完成路线 B 的基础上，继续深入**：

### 第 8 步（20 分钟）：学习监督者机制和目标对齐
📄 **文件**：`开发总纲_人类可读版.md`  
📍 **范围**：第 31-112 行（"目标对齐监督者"整章）  
🎯 **获得**：理解"为什么会有检查点，检查什么"

```
学到的内容：
- 监督者的核心工作：拿着根目标，问每个节点"你的产出能回到根目标吗？"
- 四种嫁接事故及防止机制（树根嫁接、树干嫁接、树枝嫁接、树梢嫁接）
- goal_trace：从每个节点反向追溯到根目标的完整链路
- 8 个检查点（cp_0 - cp_7）各自校验什么
```

### 第 9 步（15 分钟）：学习 v3.5 优化方案（了解总纲如何迭代）
📄 **文件**：`开发总纲_V3.5_优化方案.md`  
📍 **范围**：第 1-50 行（5 个 P0 级修复）+ 第 85-145 行（5 个 P1/P2 级修复摘要）  
🎯 **获得**：理解总纲从 v3.4 → v3.5 → v3.6 怎么改进的

```
学到的内容：
- 监督者独立性问题（S1）：自己审核自己永远通过 → 改为自动化校验
- 递归从未执行（S2）：Phase 5 只写描述，不产出递归产物 → 改为强制产出
- 接口不一致（P4-1）：更新设计时没有更新接口约束 → 改为每次 spec 更新必须重跑 Phase 4
- 验证数据虚假（P7-2）：写了"400 章"但实际只有 70 章 → 改为自动化内容校验
- 产物无版本化（S4）：旧版本被覆写 → 改为版本化归档
```

### 第 10 步（15 分钟）：学习 orchestrator.js 和调度器
📄 **文件**：`orchestrator.js`  
📍 **范围**：第 1-100 行（配置 + 函数定义）  
🎯 **获得**：理解"一句话输入"怎么变成"自动化执行"的

```
学到的内容：
- orchestrator.js 是任务初始化器，不是持续监控器
- 它做的事：创建任务文件 → 写入任务队列 → 等待三个 IDE 的 Agent 自动拿任务
- 两种模式：初始化模式（异步，Agent 轮询接管）+ 同步等待模式（调试用）
- 三个 IDE 的轮询机制和触发信号
```

### 第 11 步（15 分钟）：学习 Agent 协作模板（了解实际操作指导）
📄 **文件**：`Agent子节点沟通模板.md`  
📍 **范围**：第 1-50 行（全局问题陈述）+ 第 334-470 行（模板 10-14）  
🎯 **获得**：理解 Agent 之间怎么沟通，怎么协作

```
学到的内容：
- 模板 10：Agent 自主执行协议（不要每步等用户确认）
- 模板 11：Agent 间协作协议（通过协作目录直接对接）
- 模板 12：三 IDE 流水线调度器使用指南
- 模板 13：Codex 全自动自问自答协议（怎么做 Phase 1）
- 模板 14：全自动流水线调度协议（包含监督者）
```

### 第 12 步（10 分钟）：浏览 CHANGELOG（了解演进历史）
📄 **文件**：`CHANGELOG.md`  
📍 **范围**：第 111-201 行（v3.6.0 / v3.6.1 / v3.6.2 的变更说明）  
🎯 **获得**：理解最近的改进是什么，有什么已知的尚待完成的工作

```
学到的内容：
- v3.6.0：新增 Phase 门禁引擎（解决"跳步"问题）
- v3.6.1：实现 gate_runner.py（门禁从纸面变代码）
- v3.6.2：cp_0 增补意图覆盖度检查（防止遗漏用户需求）
- v3.6.3：照总纲补齐 cp_0-cp_7 全八检查点
- v3.6.4：总纲技能化（无断点运行）
- 仍待完成：gate_runner 未嵌入主流程，仍需执行者主动调用
```

**✅ 深入研究完成！** 现在你是"总纲专家"了，可以：
- 理解任何一个设计决策的原因
- 修改或优化总纲
- 教导其他 Agent 怎么用这个系统
